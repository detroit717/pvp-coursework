Характеристика предметной области

Система автоматизации платных дорожных пунктов (ПВП)

Назначение

Автоматизация процессов регистрации проезда транспортных средств через пункты взимания платы, расчёта стоимости, списания средств и контроля штрафов.

Участники

• Водители — регистрируются, пополняют баланс, совершают поездки
• Администратор — управляет справочниками, тарифами, просматривает статистику
• Система — автоматически рассчитывает тарифы, списывает средства, налагает штрафы

Бизнес-процессы

1. Регистрация водителя и транспортного средства
2. Проезд ТС через полосу ПВП
3. Расчёт стоимости по тарифу
4. Списание средств (баланс / транспондер)
5. Автоматический штраф при неоплате
6. Пополнение баланса
7. Аналитика и отчётность

Ключевые сущности

Водители, ТС, ПВП, полосы, тарифы, транзакции, транспондеры, штрафы

Диаграмма физической модели данных

SQL-типы, ограничения, индексы, триггеры

Таблицы (PostgreSQL 16)

CREATE TABLE transactions (
  id_transaction SERIAL PRIMARY KEY,
  id_point INTEGER NOT NULL REFERENCES payment_points,
  id_lane INTEGER NOT NULL REFERENCES lanes,
  id_vehicle INTEGER NOT NULL REFERENCES vehicles,
  amount NUMERIC(10,2) NOT NULL CHECK (amount >= 0),
  status VARCHAR(20) DEFAULT 'успешно'
    CHECK (status IN ('успешно','неоплата')),
  datetime TIMESTAMP DEFAULT NOW()
);

Индексы

CREATE INDEX idx_trans_date
  ON transactions(datetime);
CREATE INDEX idx_trans_point
  ON transactions(id_point);
CREATE INDEX idx_veh_plate
  ON vehicles(plate_number);

Ограничения CHECK

CHECK (personal_balance >= 0)     -- drivers
CHECK (amount >= 0)               -- transactions
CHECK (amount > 0)                -- fines
CHECK (lanes_count > 0)           -- payment_points
CHECK (status IN (...))           -- транзакции
CHECK (payment_status IN (...))   -- штрафы

Триггер

CREATE OR REPLACE FUNCTION auto_fine_for_non_payment()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
BEGIN
  IF NEW.status = 'неоплата' THEN
    INSERT INTO fines (...)
    VALUES (NEW.id_vehicle, NEW.id_driver,
            500.00, 'неоплачен', ...);
  END IF;
  RETURN NEW;
END;
$$;

CREATE TRIGGER trg_auto_fine
AFTER INSERT ON transactions
FOR EACH ROW EXECUTE FUNCTION
  auto_fine_for_non_payment();

Примеры запросов к БД

Разбор 2-х характерных запросов

Запрос 1: Итоговая сумма по водителям

SELECT d.full_name,
       COUNT(t.id_transaction) AS trips,
       SUM(t.amount) AS total_spent
FROM drivers d
JOIN vehicles v ON v.id_driver = d.id_driver
JOIN transactions t ON t.id_vehicle = v.id_vehicle
WHERE t.status = 'успешно'
  AND t.datetime >= NOW() - INTERVAL '30 days'
GROUP BY d.id_driver, d.full_name
ORDER BY total_spent DESC
LIMIT 10;

Аналитика: ТОП-10 водителей по сумме за 30 дней

Запрос 2: Загрузка полос ПВП

SELECT pp.name AS point_name,
       l.lane_number,
       COUNT(t.id_transaction) AS pass_count,
       COALESCE(SUM(t.amount), 0) AS revenue
FROM payment_points pp
JOIN lanes l ON l.id_point = pp.id_point
LEFT JOIN transactions t
  ON t.id_lane = l.id_lane
  AND t.datetime >= NOW() - INTERVAL '7 days'
GROUP BY pp.id_point, pp.name,
         l.id_lane, l.lane_number
ORDER BY pp.name, l.lane_number;

Описание

Триггерная функция срабатывает после вставки каждой транзакции. Если статус = 'неоплата', автоматически создаётся запись в таблице fines со штрафом 500.00₽ со статусом 'неоплачен'. Если оплата успешна — никаких действий.

Язык: PL/pgSQL
Тип: AFTER INSERT, FOR EACH ROW
